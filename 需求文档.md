# 口腔筛查登记小程序 - 产品需求文档

> 版本：V1.1.150 | 更新：2026-03-21
> 面向：中国红十字会天津口腔服务队
> 架构：微信小程序（前端）+ Django REST API（后端，本项目）

---

## 决策日志（已确认，不再改动）

| # | 决策点 | 结论 |
|---|--------|------|
| 1 | 患者身份证字段 | ✅ 保留存储 |
| 2 | 病历字段结构化 | ✅ 叙述性文字用 TextField，口腔检查结构化字段合并进 MedicalRecord |
| 3 | 旧 Patient 数据 | ✅ 能保留的保留，symptoms→chief_complaint 等字段迁移 |
| 4 | 智能客服/语音/AI | ⏸ 暂不开发 |
| 5 | 数据库选型 | ✅ 开发 SQLite / 生产 PostgreSQL |
| 6 | JSON vs 关系表 | ✅ 固定选项多选用 M2M 表；动态结构用 JSONField |
| 7 | 诊断字段 | ✅ 每次就诊一份，医生综合报告，TextField |
| 8 | 牙位记录 | ✅ 独立 `ToothFinding` 表，FDI 两位数编码 |
| 9 | 医生擅长专业 | ✅ M2M 中间表 `DoctorSpecialty`，7个专业选项，后续可扩展 |
| 10 | 病历照片 | ✅ JSONField 存路径数组，合并进 MedicalRecord，上限20张 |
| 11 | 病历相关表合并 | ✅ MedicalRecord + OralExamination 合并为一张表，RecordPhoto 废弃 |
| 12 | 医生执业地点 | ✅ 当前工作医院名称，CharField，不关联 Station |
| 13 | HL7 扩展字段策略 | ✅ 需查询的单独列（visit_no/visit_type），纯存档的进 JSONField（observations/billing/extra） |
| 14 | visit_no 生成规则 | ✅ 按日期重置，格式 `YYYYMMDD-NNN`，当天第几条，如 `20260321-001` |
| 15 | MedicalRecord 人员字段 | ✅ doctor/volunteer 均指向 User，语义够用，详情按需关联 Doctor/Volunteer Profile |
| 16 | MedicalRecord.activity | ✅ null=True，旧数据迁移时留空 |
| 17 | ToothFinding 同牙多问题 | ✅ 多行存储，同一颗牙不同 finding_type 各一行，无 unique_together 约束 |
| 18 | ActivityParticipant 角色 | ✅ 严格按 Django Group 区分，医生只能以 doctor 身份参与，不能跨角色 |
| 19 | MedicalHistory 创建时机 | ✅ 自动创建，Patient 保存时 get_or_create，有则更新无则创建 |

## 一、用户角色

| 角色 | Django Group | 说明 |
|------|-------------|------|
| 管理员 | ADMIN | 用户审批、活动管理、数据统计 |
| 医生 | DOCTOR | 查看患者、确认诊断、工作台 |
| 志愿者 | VOLUNTEER | 患者登记、参与活动（需新增 Group） |
| 患者 | PATIENT | 查询自己的筛查记录 |

## 二、数据库模型（定稿）

### ER 关系图

```
User（Django 内置）
 ├── Doctor（OneToOne）
 │    └── DoctorSpecialty（M2M 中间表）
 └── Volunteer（OneToOne）

Patient
 ├── MedicalHistory（OneToOne）   患者既往病史，稳定不变
 └── MedicalRecord（FK，多条）    每次就诊一条
      └── ToothFinding（FK，多行） 每颗问题牙一行

Activity
 └── ActivityParticipant（activity + user + role）

Station（独立，不关联 Doctor）
```

### 各表字段

#### `Patient`

| 字段 | 类型 | 说明 |
|------|------|------|
| user | OneToOne(User, null=True) | 可无账号，志愿者代录 |
| real_name | CharField(20) | 真实姓名（HL7 PID） |
| id_card | CharField(18, blank) | 身份证号（HL7 PID） |
| gender | CharField M/F/U | 性别（HL7 PID） |
| birth_date | DateField(null) | 出生日期（HL7 PID），前端计算年龄 |
| mobile | CharField(20) | 联系电话（HL7 PID） |
| address | CharField(100, blank) | 地址（HL7 PID） |
| extra | JSONField(default=dict) | 不常查询的扩展字段（HL7 PID 补充） |
| created_by | FK(User, null) | 登记志愿者 |
| created_at | DateTimeField | 自动 |
| status | BooleanField(True) | 保留旧字段 |
| ~~symptoms~~ | — | 迁移至 MedicalRecord.chief_complaint |
| ~~assignedDoctorId~~ | — | 迁移至 MedicalRecord.doctor |
| ~~admitDate~~ | — | 迁移至 MedicalRecord.check_date |

> `extra` 结构示例：`{"ethnicity": "汉族", "marital_status": "已婚"}`

#### `MedicalRecord`（每次就诊一条）

**基本信息（HL7 PV1）**

| 字段 | 类型 | 说明 |
|------|------|------|
| patient | FK(Patient) | |
| activity | FK(Activity, null=True) | 可为空，旧数据迁移时留空 |
| volunteer | FK(User) | 录入志愿者（指向 User） |
| doctor | FK(User, null=True) | 确认医生（指向 User，需要详情时关联 Doctor Profile） |
| check_date | DateField | 检查日期 |
| visit_no | CharField(unique=True) | 就诊流水号，按日期重置，如 `20260321-001`（HL7 PV1），保存时自动生成 |
| visit_type | CharField choices | 就诊类别：outpatient/charity/referral（HL7 PV1） |

**叙述性文本（不统计）**

| 字段 | 说明 |
|------|------|
| chief_complaint | 主诉 |
| present_illness | 现病史 |
| past_history_note | 本次相关既往史补充 |
| auxiliary_exam | 辅助检查描述（X光/CBCT） |
| treatment_plan | 治疗计划 |
| informed_consent | 知情同意 |
| treatment_record | 治疗过程记录 |
| followup_notes | 复诊医嘱 |

**口腔检查结构化（统计用）**

| 字段 | 类型 | 说明 |
|------|------|------|
| face_symmetry | BooleanField(True) | 面部对称 |
| has_swelling | BooleanField(False) | 有无肿胀 |
| mouth_opening | CharField choices | normal / mild_limit / severe_limit |
| extraoral_note | TextField(blank) | 口外其他 |
| has_periodontal | BooleanField(False) | 牙周问题 |
| mucosa_normal | BooleanField(True) | 黏膜正常 |
| mucosa_note | TextField(blank) | 黏膜异常描述 |
| intraoral_note | TextField(blank) | 口内其他 |
| diagnosis | TextField(blank) | 医生综合诊断报告 |

**观察结果与财务（HL7 OBX / FT1）**

| 字段 | 类型 | 说明 |
|------|------|------|
| observations | JSONField(default=list) | 检查观察结果（HL7 OBX），结构不固定存 JSON |
| billing | JSONField(null=True) | 财务收费信息（HL7 FT1），纯存档 |

> `observations` 结构示例：`[{"code": "LOINC-xxx", "name": "检查项目", "value": "...", "unit": "...", "reference": "正常", "flag": "N"}]`
> `billing` 结构示例：`{"items": [{"name": "检查费", "amount": 0}], "total": 0}`

**照片与状态**

| 字段 | 类型 | 说明 |
|------|------|------|
| photos | JSONField(default=list) | 照片路径数组，上限20张 |
| doctor_confirmed | BooleanField(False) | 医生是否已确认 |
| confirmed_at | DateTimeField(null) | 确认时间 |
| created_at | DateTimeField | 自动 |
| updated_at | DateTimeField | 自动 |

#### `ToothFinding`（FK MedicalRecord，每颗问题牙一行）

| 字段 | 类型 | 说明 |
|------|------|------|
| record | FK(MedicalRecord) | |
| tooth_number | PositiveIntegerField | FDI编码，如 16=右上第一磨牙 |
| finding_type | CharField choices | caries / missing / filling / crown / other |
| note | CharField(100, blank) | 备注 |

> FDI编码：第一位象限1-4，第二位牙序1-8，共32颗

## 三、API 接口规划

### 用户与认证
```
POST /api/auth/login/
POST /api/auth/logout/
GET  /api/user/info/
PUT  /api/user/update/
```

### 活动管理
```
GET    /api/activities/
POST   /api/activities/
GET    /api/activities/{id}/
PUT    /api/activities/{id}/
DELETE /api/activities/{id}/
POST   /api/activities/{id}/join/
GET    /api/activities/{id}/stats/
```

### 患者与病历
```
POST /api/patients/
GET  /api/patients/?activity=&name=
GET  /api/patients/{id}/
PUT  /api/patients/{id}/
GET  /api/patients/{id}/history/
PUT  /api/patients/{id}/history/
POST /api/patients/{id}/records/
GET  /api/patients/{id}/records/
GET  /api/records/{id}/
PUT  /api/records/{id}/
POST /api/records/{id}/photos/
```

### 医生工作台
```
GET /api/doctor/records/?date=&activity=

```

```
GET  /api/stations/

```

### 管理后台
