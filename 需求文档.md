### 2.2 字段设计详情

#### `Patient`（合并扩展）

```
# 保留旧字段（迁移兼容）
user            OneToOne(User, null=True)   可无账号（志愿者代录）
mobile          CharField(20)
address         CharField(40, blank=True)
status          BooleanField(default=True)

# 旧字段迁移去向
symptoms        → MedicalRecord.chief_complaint（首次病历迁移）
assignedDoctorId → MedicalRecord.doctor（有病历时关联）
admitDate       → MedicalRecord.check_date
profile_pic     → 废弃

# 新增字段
real_name       CharField(20)
id_card         CharField(18, blank=True)   ✅ 已确认保留
gender          CharField  M/F/U
birth_date      DateField(null=True)        前端根据此字段计算并显示年龄
created_by      FK(User, null=True)         登记志愿者
created_at      DateTimeField(auto_now_add)
```

#### `MedicalHistory`（既往病史，OneToOne Patient）

```
patient           OneToOne(Patient)
# 结构化布尔（供统计）
hypertension      BooleanField    高血压
heart_disease     BooleanField    心脏病
hyperglycemia     BooleanField    高血糖
hypoglycemia      BooleanField    低血糖
coagulation       BooleanField    凝血障碍
# 自由文本
medications       TextField(blank)  用药情况
allergies         TextField(blank)  过敏史
other_history     TextField(blank)  其他既往史
```

#### `MedicalRecord`（病历主表，每次就诊一条）

```
patient           FK(Patient)
activity          FK(Activity)
volunteer         FK(User, related='volunteered')     录入志愿者
doctor            FK(User, null=True, related='diagnosed')  负责医生
check_date        DateField

# 自由文本字段（叙述性，不做统计）
chief_complaint   TextField    主诉（如"右下后牙自发痛3天"）
present_illness   TextField    现病史
past_history_note TextField    本次就诊相关既往史补充
auxiliary_exam    TextField    辅助检查（X光/CBCT所见）
treatment_plan    TextField    治疗计划
informed_consent  TextField    知情同意
treatment_record  TextField    治疗过程记录
followup_notes    TextField    复诊医嘱

# 状态
doctor_confirmed  BooleanField(default=False)
confirmed_at      DateTimeField(null=True)
created_at        DateTimeField(auto_now_add)
updated_at        DateTimeField(auto_now)
```

#### `OralExamination`（口腔检查结构化子表，OneToOne MedicalRecord）✅ 方案C

```
record            OneToOne(MedicalRecord)

# 口外检查
face_symmetry     BooleanField(default=True)   面部对称
has_swelling      BooleanField(default=False)  有无肿胀
mouth_opening     CharField choices:
                    normal      正常
                    mild_limit  轻度受限
                    severe_limit 重度受限
extraoral_note    TextField(blank)  口外其他

# 口内检查（核心统计字段）
caries_count      PositiveIntegerField(default=0)  龋齿颗数
missing_count     PositiveIntegerField(default=0)  缺牙颗数
has_periodontal   BooleanField(default=False)      牙周问题
mucosa_normal     BooleanField(default=True)       黏膜正常
mucosa_note       TextField(blank)                 黏膜异常描述
intraoral_note    TextField(blank)                 口内其他

# 诊断归类（供统计）待确认选项 🔲
diagnosis_code    CharField choices（见下方待确认B）
diagnosis_note    TextField    诊断详细描述
```

## 三、ER 关系图

```
User (Django内置)
 ├── Doctor (OneToOne)
 │    ├── DoctorSpecialty (M2M) → specialty choices
 │    └── work_locations (M2M) → Station
 └── Volunteer (OneToOne)

Patient
 ├── MedicalHistory (OneToOne)   ← 既往病史，稳定
 └── MedicalRecord (FK, 多条)    ← 每次就诊
      ├── OralExamination (OneToOne)  ← 结构化检查结果
      └── RecordPhoto (FK, 多张)      ← 面部/X光照片

Activity
 └── ActivityParticipant (M2M → User)

Station ← Doctor.work_locations
```

## 四、API 接口规划

### 4.1 用户与认证
```
POST /api/auth/login/
POST /api/auth/logout/
GET  /api/user/info/        # 返回角色+基本信息
PUT  /api/user/update/
```

### 4.2 活动管理
```
GET    /api/activities/
POST   /api/activities/
GET    /api/activities/{id}/
PUT    /api/activities/{id}/
DELETE /api/activities/{id}/
POST   /api/activities/{id}/join/
GET    /api/activities/{id}/stats/
```

### 4.3 患者与病历
```
POST /api/patients/
GET  /api/patients/?activity=&name=
GET  /api/patients/{id}/
PUT  /api/patients/{id}/

GET  /api/patients/{id}/history/       # 既往病史
PUT  /api/patients/{id}/history/       # 更新既往病史

POST /api/patients/{id}/records/       # 新建病历
GET  /api/patients/{id}/records/       # 历史病历列表
GET  /api/records/{id}/                # 病历详情
PUT  /api/records/{id}/                # 更新病历（含口腔检查）
POST /api/records/{id}/photos/         # 上传照片
```

### 4.4 医生工作台
```
GET  /api/doctor/records/?date=&activity=   # 待确认病历列表
PUT  /api/doctor/records/{id}/confirm/      # 确认诊断
GET  /api/doctor/stats/
```

### 4.5 服务站
```
GET  /api/stations/
POST /api/stations/
GET  /api/stations/{id}/
PUT  /api/stations/{id}/
```

### 4.6 管理后台
```
GET  /api/admin/users/
POST /api/admin/users/{id}/approve/
GET  /api/admin/stats/
```
